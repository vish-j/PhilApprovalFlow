Interface ICanSetUser

Defines operations to set the user context for an approval flow. This interface is the entry point for interacting with an approval flow and must be used before performing any actions on the workflow.

ResetTransitions(string)

Resets all transitions to a default "Awaiting Decision" state with optional comments.

Declaration
ICanAction ResetTransitions(string comments = null)
Parameters
Type Name Description
string comments

Optional comments to associate with the reset transitions explaining why they were reset. This value can be null.
Returns
Type Description
ICanAction

This instance to allow method chaining.
Remarks

This method clears any previous decisions (approvals, rejections, etc.) on all transitions and sets them back to the initial awaiting state. This is useful when a workflow needs to be restarted due to changes in the underlying entity or process requirements.

Examples
// Reset all transitions with a comment
workflow.ResetTransitions("Process restarted due to requirement changes");

// Reset all transitions without a comment
workflow.ResetTransitions();
