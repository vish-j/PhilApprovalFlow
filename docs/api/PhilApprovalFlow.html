Namespace PhilApprovalFlow

Classes

ApprovalEngineExtensions

Flow

Provides extension methods to attach approval workflow functionality to entities. This is the main entry point for using the PhilApprovalFlow framework.

PAFNotification

Interfaces

IApprovalFlow<T>

Defines an entity that can have an approval workflow attached.

ICanAction

Defines the operations that can be performed on an approval flow after setting the user context. This interface provides the fluent API for interacting with the approval workflow.

ICanSetUser

Defines operations to set the user context for an approval flow. This interface is the entry point for interacting with an approval flow and must be used before performing any actions on the workflow.

IPAFNotification

Defines a notification generated from an approval flow transition. Notifications can be used to send emails, create tasks, or trigger other notification systems.

IPAFTransition

Defines a transition within an approval workflow. A transition represents a single approval step with its associated metadata.
